# Студенческая интерактивная карта МИИГАиК

Это учебный проект по предмету «Практикум по языкам программирования».

Проект представляет собой интерактивную карту для студентов МИИГАиК. На сайте можно посмотреть полезные места рядом с университетом и общежитиями: корпуса, метро, кафе, магазины, аптеки, копицентры и другие объекты.

Карта сделана через JavaScript API Яндекс.Карт.

## Что делает проект

- показывает объекты на настоящей интерактивной карте
- позволяет выбрать район карты
- фильтрует объекты по категориям
- ищет объект по названию
- показывает карточку выбранного объекта
- выводит адрес, описание и категорию объекта
- загружает данные из JSON-файла

```text
project/

├─ css/

│  └─ script.js
├─ data/
│  └─ objects.json
```

- `index.html` - главная страница сайта
- `css/styles.css` - внешний вид сайта
- `js/script.js` - загрузка JSON, поиск, фильтры, выбор объекта, маркеры на карте
- `data/objects.json` - данные объектов
- `backend/app.py` - Flask API для локального backend-варианта
- `backend/requirements.txt` - зависимости backend-части

## Данные объектов

Объекты хранятся в файле `data/objects.json`.

У каждого объекта есть поля с названием, категорией, адресом, описанием, координатами и районом.

Поле `area` нужно для переключения районов:

- `miigaik` - район МИИГАиК
- `studak` - район общежития №2

## Версия для GitHub Pages

Основная версия сайта работает как обычная статическая страница.

GitHub Pages открывает `index.html`, подключает JavaScript API Яндекс.Карт через CDN, а JavaScript загружает данные из файла:

```text
data/objects.json
```

То есть для опубликованной версии не нужен запущенный сервер Flask.

## Почему Flask не используется на GitHub Pages

GitHub Pages умеет показывать статические сайты, но не запускает Python и Flask-сервер.

Поэтому опубликованная версия сайта работает через обычный JSON-файл. JavaScript сам читает `data/objects.json` и показывает объекты на карте Яндекса.

Для работы карты нужен API-ключ Яндекс.Карт. На GitHub Pages этот ключ должен быть разрешён для домена опубликованного сайта.

Flask оставлен отдельно как backend-вариант для локальной проверки API.

## Как запустить статическую версию локально

Из корня проекта:

```bash
python3 -m http.server 8000
```

Потом открыть в браузере:

```text
http://localhost:8000/
```

## Как запустить Flask backend локально

Перейти в папку backend:

```bash
cd backend
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить backend:

```bash
python3 app.py
```

После запуска Flask API будет доступен локально, например:

```text
http://127.0.0.1:5000/api/objects
```
